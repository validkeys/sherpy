#### 3. **Type Safety: Unsafe type casting in ProjectHeader**
**File:** `packages/web/src/components/project/project-header.tsx:102`

```tsx
<Badge variant={getPipelineStatusVariant(pipelineStatus) as "default" | "secondary"}>
```

**Issue:** Manual type assertion indicates type mismatch. Badge variant type is too narrow.

**Fix:** Either update `getPipelineStatusVariant` return type or handle the Badge component properly:
```tsx
function getPipelineStatusVariant(status: PipelineStatus): "default" | "secondary" {
  // More specific mapping
  if (status === "intake" || status === "archived") return "secondary";
  return "default";
}
```

**Issue:** Uses native `<a>` tag which causes full page reload instead of client-side navigation.

### 🟢 LOW Priority

#### 5. **Code Duplication: Status mapping repeated across files**
**Files:**
- `packages/web/src/components/project/project-header.tsx:22-41` (variant map)
- `packages/web/src/components/project/project-header.tsx:46-65` (format map)
- `packages/web/src/components/inbox/pipeline-status-badge.tsx` (likely has similar mapping)

**Issue:** Pipeline status formatting logic duplicated across components.

**Recommendation:** Create a shared utility:
```tsx
// packages/web/src/lib/pipeline-status-utils.ts
import type { PipelineStatus } from "@sherpy/shared";

export const PIPELINE_STATUS_CONFIG: Record<PipelineStatus, {
  label: string;
  color: string;
  variant: "default" | "secondary";
}> = {
  intake: { label: "Intake", color: "slate", variant: "secondary" },
  // ... rest of statuses
};

export function formatPipelineStatus(status: PipelineStatus): string {
  return PIPELINE_STATUS_CONFIG[status]?.label || status;
}
```

#### 6. **Performance: Promise created on every render**
**File:** `packages/web/src/pages/project-detail.tsx:91`

```tsx
const projectPromise = useMemo(() => api.getProject(projectId), [api, projectId]);
```

**Issue:** While `useMemo` prevents recreation, the `api` object changes on every auth token change (from `use-api.ts`).

**Impact:** Low - only affects re-fetches when auth token updates.

**Recommendation:** Consider using a proper data fetching library like TanStack Query in future milestones for caching/invalidation.

#### 7. **UX: No loading indication for date formatting**
**File:** `packages/web/src/components/project/project-header.tsx:73-77`

**Issue:** `toLocaleDateString` can be slow on initial render. Minor issue.

**Recommendation:** Fine as-is for now, but consider memoization if performance becomes an issue.

