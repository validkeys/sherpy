# "@effect/sql Model.Class with makeRepository"

## Overview

Demonstrates the `Model.Class` pattern from `@effect/sql` for defining domain entities with variant schemas (select, insert, update, json). Combined with `Model.makeRepository`, it generates a type-safe CRUD repository with insert, update, findById, and delete operations — all validated through Effect Schema.

## Code Example

```typescript
import { Model, SqlClient } from "@effect/sql"
import { Effect, Option, Schema } from "effect"

export class Project extends Model.Class<Project>("Project")({
  id: Model.Generated(Schema.String),
  slug: Schema.String,
  name: Schema.NonEmptyTrimmedString,
  description: Schema.optional(Schema.String),
  pipelineStatus: Schema.Literal(
    "intake", "gap-analysis", "business-requirements",
    "technical-requirements", "style-anchors", "implementation-planning",
    "plan-review", "architecture-decisions", "delivery-timeline",
    "qa-test-plan", "summaries", "active-development", "completed", "archived"
  ),
  tags: Model.JsonFromString(Schema.Array(Schema.String)),
  priority: Schema.Literal("low", "medium", "high").pipe(
    Schema.withConstructorDefault(() => "medium" as const)
  ),
  createdAt: Model.DateTimeInsert,
  updatedAt: Model.DateTimeUpdate,
}) {}

const program = Effect.gen(function* () {
  const repo = yield* Model.makeRepository(Project, {
    tableName: "projects",
    idColumn: "id",
    spanPrefix: "ProjectRepository",
  })

  const created = yield* repo.insert(
    Project.insert.make({ slug: "my-project", name: "My Project" })
  )

  const found = yield* repo.findById(created.id)
  if (Option.isSome(found)) {
    yield* repo.update(
      Project.update.make({ ...found.value, name: "Updated Project" })
    )
  }

  yield* repo.delete(created.id)
})
```

## What This Demonstrates

- **Model.Class** - Define domain entities with variant schemas (select, insert, update, json)
- **Model.Generated** - Fields generated by the database (auto-increment IDs)
- **Model.DateTimeInsert / Model.DateTimeUpdate** - Auto-timestamp fields
- **Model.JsonFromString** - JSON columns stored as text, decoded as typed objects
- **Model.makeRepository** - Factory that generates type-safe CRUD operations
- **Schema.withConstructorDefault** - Default values for optional fields
- **Schema.Literal** - Enum-like fields with exact string matching
- **Project.insert.make() / Project.update.make()** - Constructors for variant schemas
- **Option-based findById** - Returns `Option.Option<T>` for may-not-exist queries

## When to Use

- **All domain entities** - Every table in `packages/api/src/services/*/` with a corresponding Model
- **CRUD repositories** - When you need standard insert/update/findById/delete operations
- **Shared schemas** - In `packages/shared/src/schemas/` for types shared between API and web
- **API request/response types** - Use the `json`, `jsonCreate`, `jsonUpdate` variants for API contracts

## Pattern Requirements

✓ Define all domain entities as `Model.Class<Entity>("Entity")({ fields })`
✓ Use `Model.Generated` for auto-increment or database-generated fields
✓ Use `Model.DateTimeInsert` for `created_at` and `Model.DateTimeUpdate` for `updated_at`
✓ Use `Model.JsonFromString` for JSON columns that store typed objects
✓ Create repositories via `Model.makeRepository(Entity, { tableName, idColumn, spanPrefix })`
✓ Use `Entity.insert.make()` for insert operations, `Entity.update.make()` for updates
✓ Handle `Option.None` from `findById` — never assume a record exists
✓ Place Model definitions in `packages/shared/src/schemas/` for cross-package access

## Common Mistakes to Avoid

✗ Using `Schema.Class` instead of `Model.Class` — Model.Class provides variant schemas
✗ Forgetting `Model.Generated` on auto-increment IDs — causes insert validation errors
✗ Using raw SQL when Model.makeRepository covers the operation — use the repository
✗ Not handling `Option.None` from `findById` — runtime error if you assume Some
✗ Placing Model definitions in `packages/api` only — they should be in `packages/shared`
✗ Omitting `spanPrefix` in makeRepository — breaks distributed tracing
