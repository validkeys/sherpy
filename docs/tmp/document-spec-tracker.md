## Specification Template Structure

Each specification will include:

### spec.md sections:
1. Document Overview
   - Purpose and role in workflow
   - Dependencies (what must exist before this)
   - Outputs (what depends on this)
2. File Format & Location
   - Extension and naming convention
   - Directory structure placement
3. Schema Definition
   - Complete structure (YAML/Markdown/JSONL)
   - Required vs optional fields
   - Field types and constraints
4. Validation Rules
   - Required field validation
   - Format requirements
   - Cross-field dependencies
   - Common errors to prevent
5. Generation Guidelines
   - How this document should be created
   - Data sources and transformation logic
   - Edge cases to handle
6. Usage Examples
   - When to use each section
   - Common patterns

### example.[ext]:
- Complete, realistic example
- Demonstrates all major sections/fields
- Shows proper formatting and structure
- Includes realistic content (not lorem ipsum)
